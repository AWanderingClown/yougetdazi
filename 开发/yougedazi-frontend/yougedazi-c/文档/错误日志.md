## 错误 #1：login.js 语法错误

**时间**: 2026-03-28

**错误信息**:
```
unknown: Unexpected token, expected "," (149:11)
149 |           }.bind(this),
      |            ^
```

**错误原因**: 
小程序 JavaScript 不支持在对象方法后直接使用 `.bind(this)` 语法

**错误代码**:
```javascript
success(res) {
  // ...
}.bind(this),  // ❌ 语法错误
fail() {
  // ...
}
```

**正确做法**:
```javascript
success: (res) => {  // ✅ 使用箭头函数
  // ...
  // this 会自动绑定到外层作用域
},
fail: () => {  // ✅ 使用箭头函数
  // ...
}
```

**经验总结**: 
- 小程序中回调函数使用箭头函数 `() => {}` 而不是 `function() {}`
- 箭头函数自动绑定外层 `this`，无需手动 `.bind(this)`

## 错误 #2：JSON 文件 require 失败

**时间**: 2026-03-28

**错误信息**:
```
Error: module 'mock-data/companions.json.js' is not defined
```

**错误原因**: 
微信小程序不支持直接使用 `require()` 加载 JSON 文件

**错误代码**:
```javascript
const mockData = require('../../mock-data/data.json');  // ❌ 不支持
```

**正确做法**:
1. 将 `.json` 文件改为 `.js` 文件
2. 使用 `module.exports` 导出数据

```javascript
// mock-data/data.js
module.exports = {
  data: [/* ... */]
};

// 引用时
const mockData = require('../../mock-data/data');  // ✅ 正确
```

**经验总结**:
- 小程序 require 只支持 JS 模块
- 模拟数据统一使用 `.js` 格式 + `module.exports`
- 引用时不需要写 `.js` 后缀

## 错误 #3：模拟数据使用 example.com 域名导致图片加载失败

**时间**: 2026-03-28

**错误信息**:
```
[渲染层网络层错误] Failed to load image https://example.com/avatar1.jpg
the server responded with a status of 404
```

**错误原因**: 
- `example.com` 是一个示例域名，实际上并没有这些图片资源
- 小程序会尝试加载这些 URL，导致 404 错误

**错误代码**:
```javascript
avatar: "https://example.com/avatar1.jpg",  // ❌ 无效链接
photos: [
  "https://example.com/photo1.jpg"  // ❌ 无效链接
]
```

**正确做法**:
使用本地资源路径或可靠的占位图服务

```javascript
// 方案1: 使用本地默认头像
avatar: "/assets/images/avatar-default.png",  // ✅

// 方案2: 使用占位图服务（如需在线图片）
avatar: "https://via.placeholder.com/100x100/FF69B4/FFFFFF?text=User",  // ✅
```

**经验总结**:
- 模拟数据中的图片 URL 必须真实可用
- 开发阶段推荐使用本地资源（assets 目录）
- 避免使用 `example.com` 等示例域名作为图片地址
- 可以使用占位图服务：placeholder.com、picsum.photos 等

## 开发规范总结

### 1. 数据文件规范
- 模拟数据统一放在 `mock-data/` 目录
- 使用 `.js` 格式 + `module.exports`
- 图片使用本地路径 `/assets/images/xxx.png`

### 2. 回调函数规范
- 统一使用箭头函数 `(res) => {}`
- 不使用 `.bind(this)`
- 对象方法使用 `{ success: () => {} }` 语法

### 3. 图片资源规范
- 开发阶段使用本地资源
- 不使用无效的示例 URL
- 确保图片文件实际存在于项目中
